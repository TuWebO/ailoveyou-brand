# AI Love You — Master Logo

## Purpose

This directory contains the source files for the AI Love You brand mark.

The logo is the single source of truth for every visual asset generated by the brand.

Everything else is derived from these files.

Examples:

- favicon
- app icons
- website logo
- social images
- printed material
- animations

Never edit generated assets manually.

Always modify the master logo first.

---

# Philosophy

The logo represents **connection**.

It is intentionally open to interpretation.

People may see:

- a heart
- two paths converging
- a conversation
- empathy
- collaboration
- human and AI embracing

There is no single correct interpretation.

The logo should invite curiosity.

---

# Design Goals

The logo should feel:

- Human
- Warm
- Calm
- Trustworthy
- Minimal
- Timeless

It should NOT feel:

- Futuristic
- Robotic
- Corporate
- Cold
- Aggressive
- Overly technical

---

# Construction Rules

The logo is built using only two mirrored curves.

Rules:

- Perfect bilateral symmetry
- Constant stroke width
- Rounded caps
- Rounded joins
- Smooth Bézier curves
- Minimal control points

Every point must exist for a reason.

Remove complexity whenever possible.

---

# Geometry

The logo should fit inside a square.

It should be optically centered.

The visual weight should feel balanced.

Negative space is part of the design.

Do not add decorative details.

---

# Scalability

The logo must work at:

- 16×16
- 32×32
- 64×64
- 128×128
- 512×512
- 1024×1024

If recognition is lost at 16 px,
the logo should be simplified rather than enlarged.

---

# Color

The master logo is monochrome.

Primary:

Black on white.

Secondary:

White on black.

Color variants belong to the design system and are generated separately.

---

# Animation

Animations should reinforce connection.

Good animations:

- drawing
- flowing
- breathing
- gentle appearance

Avoid:

- bouncing
- spinning
- flashing
- shaking

Motion should always feel calm.

---

# Accessibility

The logo must work:

- in monochrome
- on light backgrounds
- on dark backgrounds
- without gradients
- without shadows

It should remain recognizable even for users with reduced vision.

---

# Editing Guidelines

When editing the logo:

Ask yourself:

Does this make it simpler?

Does this improve recognition?

Does this strengthen the idea of connection?

Does this remove unnecessary complexity?

If the answer is "no",
the change probably should not be made.

# Versioning

The logo evolves.

Every significant visual change should be recorded in:

docs/decisions.md

Never overwrite history.

Explain why the change was made.

---

# Guiding Principle

The logo is not a heart.

The heart is only one possible interpretation.

The real symbol is **connection**.

Everything else is secondary.
